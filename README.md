# GD-Engine

GD-Engine是一个通过一系列结构化目标逐步构建的教育游戏引擎。本存储库的目标是为那些希望深入了解如何使用现代C++和核心渲染/物理/输入概念从零开始构建游戏引擎的人，提供一条清晰、循序渐进的学习路径。

---

## 📚 目标结构

所有详细的小目标资源都存储在 **`lessons/`** 目录中。  
每个目标都会介绍一个新的子系统或功能。  
您还可以浏览 **Git标签** ，其中每个标签都代表一个特定的课程检查点——使您能够查看引擎在该开发步骤时的确切状态。  
此外，每次提交都包含到目前为止的完整引擎状态，包括从最开始到当前正在检查的提交之间的所有目标。

---

## 🎯 引擎的目的

这个引擎并非旨在与Unity或Unreal等生产级引擎竞争。  
相反，它是一个**以学习为中心的代码库**，旨在揭示如何：

- 渲染管线是从零开始构建的。
- 物理和碰撞系统在内部运行。
- 场景图(Scene graphs)、组件(components)、用户界面系统(UI systems)和脚本扩展引擎架构(scripting expand engine architecture)。
- 音频集成。
- 实现了真正的2D和3D游戏逻辑。

每个子系统都逐步介绍，并提供详细的分解和简单的C++实现。

## ⚙️ 构建说明

你可以使用以下两种方法之一来构建引擎：

### 1. 通过脚本生成项目
- `./gen_proj_vs2022.bat` for Visual Studio 2022
- `./gen_proj_xcode.sh` for Xcode  
然后，在您喜欢的集成开发环境中打开`build`文件夹中生成的项目。

### 2. 在CMake模式下使用IDE打开仓库 (推荐)
只需在 **CLion / Visual Studio Code / Visual Studio (CMake mode)** 中打开仓库根目录，并让其自动配置即可。
